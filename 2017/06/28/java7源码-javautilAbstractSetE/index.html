综述

E表示被Set所持有的元素类型。

本类提供了Set接口的最小化实现。

在遵循Set接口的额外约束（例如不允许存在重复元素）的前提下，继承本类实现Set与继承AbstractCollection实现Collection所需做的工作是相同的。

注意本类没有重写其所继承的AbstractCollection的任何方法。本类仅仅是添加了equals(Object o)及hashCode()的实现。

本类是java集合框架中的一员。

源码

描述

比较o与abstractSet的相等性。满足如下条件返回true：o同样是一个Set，o与abstractSet长度相同，o中的每个元素都在abstractSet中存在。只要满足这些条件，即便是Set接口的不同实现之间也可判定为相等。

本方法首先检查o是否是abstractSet本身，如果是则返回true。然后检查o是否是一个set及长度是否与abstractSet相同，如果有一个为否则返回false。如果均为是，则返回containsAll((Collection) o)的结果。

比较与哈希：public int hashCode()

源码

继承关系

描述

返回abstractSet的hash code值。一个set的hash code值为其组成元素的hash code值之和（等于null的元素的hash code值被定义为0）。这样可以保证对于任意两个Set，s1及s2，只要有s1.equals(s2)，即有s1.hashCode()==s2.hashCode()，满足Object对于hashCode()及equals(Object obj)返回关系的要求。

本方法迭代abstractSet，调用abstractSet中每个元素的hashCode()，求和作为本方法的结果返回。

public boolean removeAll(Collection<?> c)

源码

继承关系

描述

移除abstractSet中被c所包含的元素（可选操作）。若c同样是一个Set，本操作实际为求差集。

本方法通过调用abstractSet及c的size()确定哪一个的长度更小。若abstractSet长度更小，则迭代abstractSet，检查迭代出的元素是否被c所包含，若包含，则调用迭代器的remove()将其从abstractSet中移除。若c的长度更小，则迭代c，调用abstractSet的remove(Object o)从abstractSet中移除所有被迭代器返回的元素。

注意，若abstractSet返回的iterator未实现remove()则会抛出UnsupportedOperationException。

若abstractSet因本方法发生了改变则返回true，反之返回false。
