综述

Queue中的E表示queue中元素的类型。

queue是一种有操作优先级的集合。

与Collection接口相比，Queue接口提供了额外的插入，取出，校验操作。在操作失败时每个方法的返回值可能为以下两种返回形式之一：

  • 如果操作失败则抛出一个异常。
  • 返回一个特定值（依操作不同或者为null或false）。此种情况实际是特指，对于insert方法而言，若queue为容量受限的queue则会有特殊的实现方式；在多数实现中，insert操作不会失败。
Throws exception Returns special value
Insert boolean add(E e); boolean offer(E e);
Remove E remove(); E poll();
Examine E element(); E peek();

通常情况下（并非强制要求），queue的优先顺序为FIFO（first-in-first-out），或者也可使用比较器设置特定的优先顺序。另一种数据结构Stack的通用优先顺序为LIFO（last-in-first-out）。

无论使用何种优先级的规则，remove()或poll()返回的都是queue的头元素。在FIFO规则的queue中，所有的新元素都会被添加至queue的尾部。其他实现可能会有不同的添加规则，所有实现必须明确指定其优先级的计算方法。

如果可以插入，offer(E e)方法插入元素，反之返回false。这与其继承的Collection接口的add(E e)有所不同。add(E e)在无法插入时会抛出一个unchecked exception。offer(E e)被用于如下情况：插入失败是正常情况而非异常，例如将元素插入固定容量的队列时很容易无法插入。

remove()及poll()用于移除并返回queue的头元素。移除哪个元素（或者说哪个元素才是queue的头元素）依queue的实现而异。remove()及poll()的差异只体现在queue为空时：remove()抛出一个异常，poll()返回null。

element()及peek()返回但并不移除queue的头元素。

Queue接口中并未定义在并发编程中很常见的阻塞队列方法。这些阻塞队列方法会等待元素的出现或者空间的空出，它们被定义在了Queue的直接子类接口BlockingQueue中。

Queue接口的实现通常禁止插入空元素（当然也有例外，例如LinkedList就允许插入空元素）。即便是在例外中，也不应该向这种Queue的实现中插入空元素。因为poll()在queue为空时会返回null，而如果queue中包含空元素，则我们无法判断poll()返回的null所表达的含义到底是queue为空还是queue返回了一个空元素。

在equals(Object o)及hashCode()的设计上，Queue接口的实现在定义相等或计算hash code值时通常不是基于元素本身的类型的，而是基于类Object的。因为元素本身的类并不总能很好的定义同一个元素在不同的队列优先级下的相等性。

本接口是java集合框架中的一员。

boolean add(E e);

源码

描述

在不超过queue容量限制的情况下，将e插入到queue中（具体插入位置由queue的优先级计算方式而定）。若插入成功则返回true,若因容量不足而插入失败则抛出异常IllegalStateException。

该方法会抛出如下异常：

  • IllegalStateException：因容量限制e此时无法被插入queue。
  • ClassCastException：e因其所属类禁止被插入queue。
  • NullPointerException：e为空且queue禁止包含空元素。
  • IllegalArgumentException：e其某些属性禁止被插入queue。

boolean offer(E e);

源码

描述

在不超过queue容量限制的情况下，将e插入到queue中（具体插入位置由queue的优先级计算方式而定）。若插入成功则返回true,若因容量不足而插入失败则返回false。当queue有容量限制时，本方法通常优于add(E e)。因为对于有容量限制的queue而言，因容量已满无法插入元素是正常情况，不需要抛出异常。

该方法会抛出如下异常：

  • ClassCastException：e因其所属类禁止被插入queue。
  • NullPointerException：e为空且queue禁止包含空元素。
  • IllegalArgumentException：e其某些属性禁止被插入queue。
