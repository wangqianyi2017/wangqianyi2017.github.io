综述

Set是一种没有重复元素的集合。更一般的说，Set中的元素e1,e2不存在如下情况：e1.equals(e2)。因此Set中最多只能有一个元素为null。顾名思义，本接口定义的是数学中Set的概念。

与其继承的超类接口Collection相比，Set接口设置了额外的规范：

  • 构造函数
  • add()
  • equals()
  • hashCode()

未做额外规范设置的其他继承自Collection的方法也会被列举在Set的源码中（与Collection相比，Set并未添加新的方法）。

按照定义，Set的构造函数创造出的Set必须不能包含重复元素。

当以容易改变的对象作为Set的元素时必须要提高警惕。Set并未定义当元素发生改变且该改变影响元素equals()方法时的情况。因此，Set禁止自包含。

一些Set的实现对其包含的元素有限制。例如，一些实现禁止包含空元素；一些实现对其元素的类型有限制。试图插入一个不合规的元素会抛出一个unchecked exception。例如NullPointerException，ClassCastException。试图查询一个不合规的元素可能会抛出一个异常，或者仅仅只是返回false。这种依实现不同不确定会抛出的异常在本接口的定义中将被标记为“可选”。

本接口是java集合框架中的一员。

查询操作：Object[] toArray();

源码

描述

返回包含set所有元素的数组。若set的实现是有序的（迭代器返回的结果有序），则返回的数组必须同样有序。

返回的数组与set之间不存在引用关系（即使set是基于数组的），数组中的元素是set中的元素的浅拷贝。

本方法是连接集合与数组之间的桥梁。

查询操作： T[] toArray(T[] a);

源码

描述

返回包含set所有元素的数组。返回数组的类型由a的类型确定。若a.length==set.size()，则将返回的结果直接填入a后返回a（若a中已有值，则原值会被替换）。若a.length>set.size()，数组索引在[0,set.size()-1]之间的元素将被set中的元素覆盖。数组索引=set.size()的元素将被置为null（若set不允许包含空元素，则本方法此时可用来计算set的长度：返回的数组第一次出现null的索引即为set的长度），数组索引>set.size()（如果有的话）的元素不变。若a.length<set.size()，则不修改a，而是以a的类型新建长度为set.size()的数组并填入set中的值，并返回新生成的数组。

若set是有序的（迭代器返回的结果有序），则返回的数组必须有同样的顺序。

同toArray()方法，本方法是连接集合与数组之间的桥梁，且返回的数组与set之间不存在引用关系（即使set是基于数组的），数组中的元素是set中的元素的浅拷贝。

修改操作：boolean add(E e);

源码

描述

本方法可选，实现类可依自身情况决定是否真的需要实现。

若e在set中不存在则将e插入到set中。更一般的说，若set不包含e2这种元素：(e==null ? e2==null : e.equals(e2))，则将e插入set。

若set已包含e，本方法不会改变set并返回false。本方法配合Set的构造函数，可保证set永远不会包括重复元素。

以上的规定不意味着在满足不重复的前提下set必须接受所有元素，set可能会拒绝添加任意特定元素，包括null，并抛出一个异常。

  • UnsupportedOperationException：set不支持add操作。
  • ClassCastException：e因为其所属的类禁止被插入set。
  • NullPointerException：e==null且set禁止包含空元素。
  • IllegalArgumentException：e因其某些属性禁止被插入set。

修改操作：boolean remove(Object o);

源码

描述

本方法可选，实现类可依自身情况决定是否真的需要实现。

移除set中符合条件的元素e：(o==null ? e==null : o.equals(e))。

若本方法改变了set则返回true，反之返回false。

一旦调用该方法后，set将不会再包含与o相等的元素。

该方法可能会抛出如下异常：

  • ClassCastException：o的类型与set不相容（依实现可选）
  • NullPointerException：o==null且set不允许空元素存在（依实现可选）
  • UnsupportedOperationException：set不支持本方法。

批量操作：boolean containsAll(Collection<?> c);

源码

描述

若set中包含c中所有元素则返回true。反之返回false。

特别的，若c同样为Set且为set的子集，则返回true。

该方法类似于contains(Object o)。

该方法可能抛出如下异常：

  • ClassCastException：c中至少一个元素的类型与set不相容（依实现可选）
  • NullPointerException：c中至少包含一个空元素且set不允许空元素存在（依实现可选）；或c==null。

批量操作：boolean addAll(Collection<? extends E> c);

源码

描述

本方法可选，实现类可依自身情况决定是否真的需要实现。

在保证set中不存在重复元素的前提下，将c中所有元素插入set。

特别的，若c同样为一个Set，本方法实际上是将set改造为了该set与c的并集。

未定义发生如下事件时的应对策略：在操作过程中c发生变化。

若set因本方法发生变化则返回true，反之返回false。

该方法会抛出如下异常：

  • UnsupportedOperationException：set不支持本方法。
  • ClassCastException：c中任意一个元素因为其所属的类禁止被插入set。
  • NullPointerException：c中任意一个元素为null且set禁止包含空元素；或c==null。
  • IllegalArgumentException：c中任意一个元素因其某些属性禁止被插入set。

该方法类似于add(E e)。

比较及哈希：boolean equals(Object o);

源码

描述

比较o与set是否相等。

如下情况判断为相等：o为Set，两个set长度相同，本set中有o中所有的元素（或者也可以反过来说，o中有set中所有的元素）。只要满足该相等条件，Set接口的不同实现之间也可判断为相等。

比较及哈希：int hashCode();

源码

描述

返回set的hash code。

set的hash code被定义为set中所有元素的hash code的和，空元素的hash code被定义为0。若s1与s2为两个set，这确保了s1.equals(s2)即有s1.hashCode()==s2.hashCode()。满足了java规范中两个对象equals(Object o)相等即要有hashCode()相等的规范。
