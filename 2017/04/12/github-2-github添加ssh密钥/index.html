发表于 2017-04-12 | 分类于 github |
字数统计 903 | 阅读时长 3

本文操作方法面向面向系统

windows7

概述

为了能将本地的代码推送至git上，必须在每次推送时自证身份，即输入账号密码。这和我们使用账号密码登陆某网站类似。每次都需要填写账号密码是很麻烦的事情，既然用浏览器登陆网站时通常都会有“记住密码”的功能，那么使用github推送代码时是否也有类似的功能呢？答案是肯定的，也就是本文的ssh密钥机制。

简单来说，密钥机制也就是我们常说的“对口令”。密钥一式两份，一份成为公钥，一份称为私钥。私钥存放在宿主机中，而公钥则分发至所有该机授权的配对机器上。一旦双方对上身份，即相当于完成了验证身份的操作。

具体到本文的实例中，私钥存储在本地机器中，而公钥则上传到github服务器的自身账号下。
