综述

Collection中的E代表集合中元素的类型。

Collection接口为集合类层级中的根接口，jdk不会提供Collection接口的直接实现类，而是会实现Collection接口的子接口（例如Set，List）。

通常意义下，所有Collection接口的实现类（实际是实现Collection接口的子接口）应提供两个“标准”的构造函数：

  • void（无参数）的构造函数，它创建一个空的集合。
  • 带有一个集合类型的参数的构造函数，该构造函数依传入参数创建一个新的集合。

实际上，带有参数的构造函数允许实现类复制任何集合类生成新的集合，而非只能复制实现类本身所实现的集合。Collection接口无法从规则上限制这一点（因为接口不能包含构造函数）。但所有jdk提供的集合类的实现均自觉遵守这一规范。

本接口包含“破坏性”方法。这些方法可以改变其所操作的集合。若某个具体的实现类不支持某个“破坏性”方法，则会抛出异常：UnsupportedOperationException。有时，即便某集合不支持某“破坏性”方法，但只要该方法不真的产生破坏集合的结果，也可以选择不抛出UnsupportedOperationException异常。例如：某不可变集合是不支持addAll(Collection)方法的，理论上当调用该方法时应抛出UnsupportedOperationException异常。但是若传入的集合为空，则也可不抛出异常。

某些Collection接口的实现可能会对其所包含的元素有所限制。例如，有些实现禁止插入null元素，有些实现则会限制元素类型。试图插入不合法的元素时会抛出一个unchecked exception。通常为NullPointerException或ClassCastException。试图查询一个不合法的元素可能会抛出异常或者返回false。

每个集合的实现类自行定义自身的同步策略。若实现类缺乏健壮的同步机制，则在多线程环境下将出现各种线程同步问题。

本接口是java集合框架中的一员。

查询操作：int size();

返回该集合包含的元素个数。若该集合包含的元素个数>Integer.MAX_VALUE，则返回Integer.MAX_VALUE。

查询操作：boolean isEmpty();

若该集合不包含元素则返回true。反之返回false。

查询操作：boolean contains(Object o);

若集合包含特定元素则返回true。更通俗的说，当且仅当该集合包含至少一个元素e，满足(o==null ? e==null : o.equals(e))，返回true

该方法可能抛出如下异常：

  • ClassCastException：传入参数o的类型与该集合不相容（依实现可选）
  • NullPointerException：传入参数o为空且该集合不允许空元素存在（依实现可选）

查询操作：Iterator iterator();

返回一个该集合的迭代器。通常情况下迭代器不保证数据有序（除非特定的实现类保证这一点）。

查询操作：Object[] toArray();

返回包含该集合所有元素的数组。若该集合是有序的（迭代器返回的结果有序），则返回的数组必须同样有序。

返回的数组中的元素应是原集合中元素的深拷贝而非其引用，这样才能随意对新返回的数组进行操作而不需担心改变原集合。

本方法是连接集合与数组之间的桥梁。

查询操作： T[] toArray(T[] a);

返回包含该集合所有元素的数组。返回数组的类型由传入参数a的类型确定。若a.length>=collection.size()，则将返回的结果直接填入a后返回a（若a中已有值，则原值会被清空）。数组下标大于集合长度的部分将被置为null。若a.length<collection.size()，则不修改a，而是以a的类型新建长度为collection.size()的数组并填入collection中的值，并返回新生成的数组。

若该集合是有序的（迭代器返回的结果有序），则返回的数组必须同样有序。

同Object[] toArray();方法，返回的数组中的元素应是原集合中元素的深拷贝而非其引用，这样才能随意对新返回的数组进行操作而不需担心改变原集合。

修改操作：boolean add(E e);

本方法可选，实现类可依自身情况决定是否真的需要实现。

若集合因插入操作而改变则返回true。反之（集合不允许包含重复元素且待插入元素在集合中已存在）则返回false。因此对于那些不能包含重复元素的集合而言，该方法也可用于判断集合是否已包含e。

支持该操作的集合可能会对插入操作做出限制。具体来说，某些集合禁止插入空元素，某些集合会限制插入元素的类型。

若某集合禁止插入元素（不是因为集合不允许包含重复元素且待插入元素在集合中已存在，而是因为其他任意原因），则该方法应抛出异常。本机制用于保护那些常量值的集合。

该方法会抛出如下异常：

  • UnsupportedOperationException：collection不支持add操作。
  • ClassCastException：e因为其所属的类禁止被插入collection。
  • NullPointerException：e为null且collection禁止包含空元素。
  • IllegalArgumentException：e因其某些特性禁止被插入collection。
  • IllegalStateException：因插入限制，此时e不能被插入collection。

修改操作：boolean remove(Object o);

本方法可选，实现类可依自身情况决定是否真的需要实现。

移除一个collection符合条件的元素e：(o==null ? e==null : o.equals(e))。若有多个e符合条件，则本方法只会移除其中一个，具体移除哪一个依具体实现而定。

若本方法改变了原集合则返回true，反之返回false。

该方法可能会抛出如下异常：

  • ClassCastException：传入参数o的类型与该集合不相容（依实现可选）
  • NullPointerException：传入参数o为空且该集合不允许空元素存在（依实现可选）
  • UnsupportedOperationException：collection不支持remove操作。

批量操作：boolean containsAll(Collection<?> c);

若该集合中包含c中所有元素则返回true。

该方法类似于collection.contains(Object o)

该方法可能抛出如下异常：

  • ClassCastException：传入参数c中至少一个元素的类型与该集合不相容（依实现可选）
  • NullPointerException：传入参数c中包含至少一个空元素且该集合不允许空元素存在（依实现可选）；或者传入参数c本身为空

批量操作：boolean addAll(Collection<? extends E> c);

本方法可选，实现类可依自身情况决定是否真的需要实现。

将c中所有元素插入该集合。本方法并未定义如下事件发生时的解决策略：在将c中的元素添加至该集合的过程中c发生变化（这也意味着如下事件的解决策略同样未定义：将一个非空集合添加至自身）。

若该集合因本方法发生变化则返回true，反之返回false。

该方法会抛出如下异常：

  • UnsupportedOperationException：collection不支持addAll操作。
  • ClassCastException：c中任意一个元素因为其所属的类禁止被插入collection。
  • NullPointerException：c中任意一个元素为null且collection禁止包含空元素；或者c为null。
  • IllegalArgumentException：c中任意一个元素因其某些特性禁止被插入collection。
  • IllegalStateException：因插入限制，此时并非c中所有元素都能被插入collection。

该方法类似于collection.boolean add(E e);

批量操作：boolean removeAll(Collection<?> c);

本方法可选，实现类可依自身情况决定是否真的需要实现。

从该集合中移除所有与c的交集元素（求差集）。

若该集合因本方法发生变化则返回true，反之返回false。

该方法可能会抛出如下异常：

  • UnsupportedOperationException：collection不支持removeAll操作。
  • ClassCastException：传入参数c中至少有一个元素的类型与该集合不相容（依实现可选）
  • NullPointerException：本集合中至少有一个元素为空且c不允许空元素存在（依实现可选）；或者传入参数c为空。

该方法类似于collection.boolean remove(Object o);

批量操作：boolean retainAll(Collection<?> c);

本方法可选，实现类可依自身情况决定是否真的需要实现。

保留该集合中与c的交集元素。

若该集合因本方法发生变化则返回true，反之返回false。

该方法可能会抛出如下异常：

  • UnsupportedOperationException：collection不支持retainAll操作。
  • ClassCastException：传入参数c中至少有一个元素的类型与该集合不相容（依实现可选）
  • NullPointerException：本集合中至少有一个元素为空且c不允许空元素存在（依实现可选）；或者传入参数c为空。

批量操作：void clear();

本方法可选，实现类可依自身情况决定是否真的需要实现。

清空该集合。该方法执行完成后集合将为空。

该方法会抛出如下异常：

  • UnsupportedOperationException：collection不支持clear操作。

比较及哈希：boolean equals(Object o);

比较o与本集合是否相等（若具体的实现类未做特殊的设置，该方法将调用Object类的equals方法，即以引用地址判断是否相等），若相等则返回true，反之返回false。

jdk提供的直接继承Collection接口的子接口有3个：Queue，List，Set。其中List，Set均对equals方法有要求，它们要求其实现类在比较两个集合相等时不是用默认Object对象比较引用是否相同，而是根据集合具体的特点，比较“值”是否相等。因此，当程序员使用Queue接口，或者是自己直接实现Collection接口时，一定要考虑好这个equals方法的设计，通常情况下，我们比较两个集合的相等性时，都不希望比较的是其引用地址是否相等，因为这通常是毫无意义的。

通常意义上，我们所认为的相等都是对称的，即若a.equals(b)，则必应有b.equals(a)。

在jdk的集合类设计中，规定了大前提：list只能与list相等，set只能与set相等。因此，我们无法写出一个类同时实现Set及List接口，因为若这样的实现类可以存在，这意味着有一种集合既是List又是Set，与上文的逻辑相悖。

比较及哈希：int hashCode();

返回该集合的hash code值（若具体的实现类未做特殊的设置，该方法将调用Object类的hashCode方法）。

必须注意的是，任意想要重写Object.equals(Object)方法的类都必须重写Object.hashCode()。具体来说，若c1.equals(c2)，则同样应有c1.hashCode()==c2.hashCode()。
