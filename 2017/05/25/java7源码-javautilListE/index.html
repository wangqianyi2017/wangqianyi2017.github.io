综述

List（列表）又名sequence（序列）。是一种有序的集合。

使用者能精确的知道List中每个元素的位置，能通过元素在List中的索引（即在List中的位置）访问或搜索元素。

不同于Set，List允许重复元素，更通俗的说，若设e1,e2为List中的两个元素，则允许e1.equals(e2)。特别的，若List中允许空元素，则其也可允许复数个空元素。

如果有人想实现一种List：禁止重复，且在试图插入重复元素时抛出运行时异常。那么很遗憾List接口不支持这种实现。

与继承的超类接口Collection相比，List接口在如下方法中规定了额外的规范：iterator(),add(E e),remove(Object o),equals(Object o),hashCode()。为了方便起见，超类接口Collection中的其他方法本接口也会再写一遍。

List接口提供了4个可用于基于位置（索引）访问元素的方法:get(int index)，set(int index, E element)，add(int index, E element)，remove(int index)。类似于java数组，List的索引以0开始。对某些实现（例如LinkedList）而言，这些操作获取结果的时间复杂度与索引值成比例（链表取到某个索引值的元素需要在链式结构中逐个向后查找）。因此在不确定List接口的具体实现时，比起使用这种基于索引的访问元素的方法，更推荐使用迭代器。

List接口提供一个特殊的迭代器，名为：ListIterator。在通用Iterator所实现的功能的基础上，ListIterator还实现了：元素插入，元素替换，双向访问。List接口提供了一个方法获得这个ListIterator，同时该方法还可以指定ListIterator从一个特定的索引值开始。

List接口提供了两个方法用于搜索特定的对象：indexOf(Object o)，lastIndexOf(Object o)。从性能的角度考虑，这两个方法应谨慎使用。在很多实现中，这两个方法采用代价昂贵的顺序查找（linear search）。

List接口提供了两个方法用于在任意位置高效的插入或删除复数个元素。

注意：当List允许包含自身以作为自身的元素时，这种List中的equals(Object o)及hashCode()方法往往难于定义。

一些List的实现会对它们所能包含的元素有所限制。例如，一些实现禁止包含空元素，一些实现对其所能包含的元素的类型有限制。试图插入一个不合规的元素会抛出一个unchecked exception，例如NullPointerException，ClassCastException。依具体实现不同试图查询一个不合规的元素可能会抛出一个异常，或者仅仅只是返回false。

本接口是java集合框架中的一员。

比较及哈希：boolean equals(Object o);

源码

描述

比较o与list是否相等。当且仅当满足如下所有情况时返回true：

  • o为一个List
  • o与list长度相同。
  • o与list所有对应位置上的元素相等(e1==null ? e2==null : e1.equals(e2))。

换句话说，当两个List在相同位置上的元素均相等，则认为二者相等。

只要满足如上相等的定义，List接口的不同实现类也可被认定为相等。

描述

视图：List subList(int fromIndex, int toIndex);

源码

描述

返回list的一部分，索引值区间：[fromIndex,toIndex)。特别的，若fromIndex==toIndex，则返回空List（不是null）。
