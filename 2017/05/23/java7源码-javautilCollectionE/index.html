综述

Collection中的E代表集合中元素的类型。

本接口为集合类层级中的根接口，jdk不会提供本接口的直接实现类，而是会实现本接口的子接口（Set，List，Queue）。

通常意义下，所有本接口的实现类（实际是实现本接口的子接口）应提供两个“标准”的构造函数：

  • void（无参数）的构造函数，它创建一个空的集合。
  • 带有一个集合类型的参数的构造函数，该构造函数依传入参数创建一个新的集合。

实际上，带有参数的构造函数允许实现类以任何集合为基础生成新的集合，而非只能以实现类本身所实现的集合为基础。本接口无法从规则上限制这一点（因为接口不能包含构造函数）。但所有jdk提供的集合的实现均自觉遵守这一规范（即只能以实现类本身所实现的集合为基础）。

本接口包含“破坏性”方法。这些方法可以改变其所属的集合。若某个具体的实现类不支持某个“破坏性”方法，则会抛出UnsupportedOperationException。有时，即便某集合不支持某“破坏性”方法，但只要该方法不产生真的破坏性的结果，也可以不抛出UnsupportedOperationException。例如：某不可变集合不支持addAll(Collection<? extends E> c)，理论上当调用该方法时应抛出UnsupportedOperationException。但是若null==c，则也可不抛出UnsupportedOperationException。

某些本接口的实现可能会对其所包含的元素有所限制。例如，有些实现禁止插入空元素，有些实现则会限制元素类型。试图插入不合法的元素会抛出一个unchecked exception。通常为NullPointerException或ClassCastException。试图查询一个不合法的元素可能会抛出异常或者返回false。

每个集合的实现类自行定义自身的同步策略。若实现类缺乏健壮的同步机制，则在并发环境下可能会有同步问题。

本接口中的许多方法是按equals方法定义的。例如contains(Object o)的api中有如下描述：

这个描述不应被理解为具体的实现类在实现该方法时，只要传入的o不为null，就一定会遍历集合中的元素e，并进行o.equals(e)比较。只要能实现o.equals(e)的效果，实现类并不需要一定调用o.equals(e)，而是可选择更优的解决方案。例如，实现类可比较两元素的hash code。

返回包含collection所有元素的数组。若collection是有序的（迭代器返回的结果有序），则返回的数组必须有同样的顺序。

查询操作： T[] toArray(T[] a);

源码

描述

返回包含collection所有元素的数组。返回数组的类型即为a的类型。若a.length==collection.size()，则将返回的结果直接填入a后返回a（若a中已有值，则原值会被覆盖）。若a.length>collection.size()，数组范围在[0,collection.size() -1]的索引将被collection对应位置的元素覆盖；数组索引=collection.size()的元素将被置为null（若collection不允许包含空元素，则本方法此时可用来计算collection的长度：返回的数组第一次出现null的索引即为collection的长度），数组中后续元素不变。若a.length<collection.size()，则不修改a，而是以a的类型新建长度为collection.size()的数组并填入collection中的值，并返回新生成的数组。

若collection是有序的（迭代器返回的结果有序），则返回的数组必须有同样的顺序。

修改操作：boolean add(E e);

源码

描述

本方法可选，实现类可依自身情况决定是否真的需要实现。

若collection因本方法而改变则返回true。反之则返回false（例如，collection不允许包含重复元素且e在collection中已存在）。因此对于那些不能包含重复元素的集合而言，该方法也可用于判断集合是否已包含e。

支持本方法的集合可能会对本方法做出限制。例如，某些集合禁止插入空元素，某些集合会限制插入元素的类型。

若某集合禁止插入元素（不是因为集合不允许包含重复元素且e在集合中已存在，而是因为其他任意原因），则该方法应抛出异常。本机制用于保护那些常量值的集合。

该方法会抛出如下异常：

  • UnsupportedOperationException：collection不支持本方法。
  • ClassCastException：e因为其所属的类禁止被插入collection。
  • NullPointerException：e==null且collection禁止包含空元素。
  • IllegalArgumentException：e因其某些属性禁止被插入collection。
  • IllegalStateException：因插入限制，此时e不能被插入collection。

修改操作：boolean remove(Object o);

源码

描述

本方法可选，实现类可依自身情况决定是否真的需要实现。

移除collection符合条件的元素e：(o==null ? e==null : o.equals(e))。若有多个e符合条件，则本方法只会移除其中一个，具体移除哪一个依具体实现而定。

若本方法改变了collection则返回true，反之返回false。

该方法可能会抛出如下异常：

  • ClassCastException：o的类型与collection不相容（依实现可选）
  • NullPointerException：o==null且collection不允许空元素存在（依实现可选）
  • UnsupportedOperationException：collection不支持本方法。

批量操作：boolean addAll(Collection<? extends E> c);

源码

描述

本方法可选，实现类可依自身情况决定是否真的需要实现。

将c中所有元素插入collection。本方法未定义如下事件发生时的解决策略：在将c中的元素添加至collection的过程中c发生变化（这也意味着如下事件的解决策略同样未定义：将一个非空集合添加至自身）。

若collection因本方法发生变化则返回true，反之返回false。

该方法会抛出如下异常：

  • UnsupportedOperationException：collection不支持本方法。
  • ClassCastException：c中任意一个元素因为其所属的类禁止被插入collection。
  • NullPointerException：c中任意一个元素为null且collection禁止包含空元素；或c==null。
  • IllegalArgumentException：c中任意一个元素因其某些属性禁止被插入collection。
  • IllegalStateException：因插入限制，此时并非c中所有元素都能被插入collection。

该方法类似于add(E e)。

比较及哈希：boolean equals(Object o);

源码

描述

比较o与collection是否相等（若具体的实现类未做特殊的设置，该方法将调用Object类的equals(Object obj)，即以引用地址判断是否相等），若相等则返回true，反之返回false。

jdk提供的直接继承本接口的子接口有3个：Queue，List，Set。其中List，Set均对equals方法有要求，它们要求其实现类的equals(Object o)不是继承自Object类，而是根据集合具体的特点，比较“值”是否相等。因此，当程序员使用Queue接口，或者是自己直接实现本接口时，一定要考虑好本方法的设计，通常情况下，我们比较两个集合的相等性时，都不希望比较的是其引用地址是否相等，因为这通常是毫无意义的。

通常意义上，我们所认为的相等都是对称的，即若a.equals(b)，则必应有b.equals(a)。

在jdk的集合类设计中，规定了大前提：list只能与list相等，set只能与set相等。因此，我们无法写出一个类同时实现Set及List接口，因为若这样的实现类可以存在，这意味着有一种集合既是List又是Set，与大前提的逻辑相悖。

比较及哈希：int hashCode();

源码

描述

返回collection的hash code值（若具体的实现类未做特殊的设置，该方法将调用Object类的hashCode()）。

必须注意的是，任意想要重写Object类equals(Object obj)的类都必须重写Object类的hashCode()。具体来说，若c1.equals(c2)，则同样应有c1.hashCode()==c2.hashCode()。
