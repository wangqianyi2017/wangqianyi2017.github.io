综述

Iterator中的E代表该迭代器返回的元素类型。

Iterator是基于Collection的，在java的集合框架中Iterator用于代替Enumeration遍历Collection中的元素。Iterator与Enumeration有如下两点不同：

  • 在迭代过程中，Iterator允许删除（仅仅是删除，添加不行）集合中的元素；而Enumeration则不能删除。
  • Iterator使用了改进了方法名（更为精简易读）。

foreach的底层实现虽然使用了Iterator，但是因为隐藏了具体的Iterator对象，因此也无法调用对应的Iterator.remove()方法。

Enumeration接口诞生于JDK1.0，而Iterator接口诞生于1.2。Iterator是Enumeration的改进，二者所定义的功能是重叠的，因此在实现上应尽可能使用Iterator。

本接口是java集合框架中的一员。

boolean hasNext();

在迭代过程中，若集合还有元素则返回true。换句话说，返回ture说明仍然可以通过Iterator.next()方法取到元素，若在本方法返回false时继续调用Iterator.next()方法，则Iterator.next()方法会抛出异常。

E next();

返回迭代器游标所指向的元素，同时游标后移一位。

该方法抛出如下异常：

  • NoSuchElementException：Iterator游标所指的位置已无元素。

void remove();

本方法可选，实现类可依自身情况决定是否真的需要实现。

移除当前游标所指元素的前一个元素。即上一次调用Iterator.next()方法所返回的元素。因此，调用Iterator.remove()之前必须调用一次Iterator.next()。调用一次Iterator.next()后多次调用Iterator.remove()是不允许的。Iterator.remove()必须与Iterator.next()一一配对。

本方法没有设计应对以下事件的对策：在迭代过程中，被迭代集合因本方法以外的原因发生了改变。

该方法抛出如下异常：

  • UnsupportedOperationException：Iterator不支持remove操作。
  • IllegalStateException：在调用本方法前没有调用与之配对的Iterator.next()方法（包含调用一次Iterator.next()后多次调用本方法的情况）。
