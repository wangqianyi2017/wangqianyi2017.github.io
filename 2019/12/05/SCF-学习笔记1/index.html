SCF-学习笔记1

发表于 2019-12-05 | 分类于 SCF | 阅读次数
字数统计 531 | 阅读时长 2

SCF是58自主研发的RPC框架，致力于在分布式环境下提供高性能、高可靠和透明化的RPC 远程调用方案。RPC是远程过程调用（Remote Procedure Call）的缩写，它是一种通过网络从远程计算机程序上请求服务，而不需要了解底层网络技术的协议。

网上找到的参考资料：58集团RPC框架SCF的设计与实践
这篇文章在SCF原理这块说的很清楚，有些概念没有整的特别清楚，等有时间专门看一下。没有查到58的SCF的全称是什么，我猜测是Server Client Factory，等之后确认再来更新。
简写一下，自己是如何搭建一个SCF。

  1. 搭建服务端S
  2. 搭建客户端C，依赖注入服务端S的jar包
  3. 配置SCF（下载scf.zip,解压）
  4. 将服务端S的jar包，lib，配置文件(scf_config，scf_log4j.xml),这四个文件放在scf/service/deploy/test(服务名)目录下
  5. 启动 ./bin/scf start 服务名（../service/deploy/服务名）
  6. 配置客户端C的配置文件，IP，端口，服务名
  7. 运行客户端C，结束

遇到的问题：
1.写服务端的时候，依赖的jar包一直都未能导入，后来查询原因，maven_settings.xml要用58的，其次要修改文件里的本地安装maven库的地址。
2.在写客户端的时候，scf.config, scf_log4j.xml这两个配置文件，要注意配置IP，端口，服务名
3.在服务器上部署SCF的时候，也要编写scf_config，scf_log4j.xml，这是服务端的配置文件，与客户端不同，不要混为一谈。在这里有一个需要注意的地方，日志配置文件有log4j/log4j2两种，需要跟SCF依赖包对应起来使用，不然就会看不见日志。

下次尝试，将服务端依赖的jar包和工程一起打成服务端的jar包后，去掉lib，是否还可以启动scf2.
